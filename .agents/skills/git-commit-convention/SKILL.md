# Git Commit Convention Skill

## Description
이 스킬은 git 커밋을 "(영어 카테고리):(한글 메시지)" 형식으로 작성하고, 작게 나누어서 자주 커밋한 후 자동으로 푸시하는 기능을 제공합니다.

## Usage
이 스킬은 다음과 같은 상황에서 유용합니다:
- 작업 중인 변경사항을 논리적인 단위로 나누어 자주 커밋하고 싶을 때
- 커밋 메시지를 일관된 형식으로 유지하고 싶을 때
- 커밋 후 자동으로 원격 저장소에 푸시하고 싶을 때

## Instructions
이 스킬을 사용하려면 다음 단계를 따르세요:

1. **변경사항 확인**: 먼저 `git status`를 실행하여 현재 작업 중인 파일을 확인합니다.
2. **파일 추가**: 변경사항을 논리적인 단위로 나누어 `git add` 명령어로 스테이지 영역에 추가합니다.
3. **커밋 메시지 작성**: "(영어 카테고리):(한글 메시지)" 형식으로 커밋 메시지를 작성합니다.
   - 영어 카테고리 예시: feat, fix, docs, style, refactor, test, chore 등
   - 한글 메시지 예시: "로그인 기능 추가", "버그 수정", "문서 업데이트" 등
4. **커밋 실행**: `git commit -m "(영어 카테고리):(한글 메시지)"` 명령어로 커밋을 생성합니다.
5. **푸시 실행**: `git push` 명령어로 원격 저장소에 푸시합니다.

## Examples
다음은 이 스킬을 사용하는 예시입니다:

```bash
# 변경사항 확인
git status

# 특정 파일 추가 (예: 로그인 기능 관련 파일들)
git add src/components/Login.js src/services/authService.js

# 커밋 실행 (영어 카테고리:feat, 한글 메시지:로그인 기능 추가)
git commit -m "feat:로그인 기능 추가"

# 원격 저장소에 푸시
git push
```

## Best Practices
1. **작게 나누기**: 하나의 기능이나 수정 사항이라도 가능한 작은 단위로 나누어 커밋하세요.
2. **일관된 카테고리 사용**: 다음과 같은 표준 카테고리를 사용하세요:
   - feat: 새로운 기능 추가
   - fix: 버그 수정
   - docs: 문서 추가/수정
   - style: 코드 스타일 변경 (세미콜론, 공백 등)
   - refactor: 코드 리팩터링
   - test: 테스트 추가/수정
   - chore: 빌드 작업, 패키지 매니저 설정 등
3. **명확한 한글 메시지**: 변경사항을 명확히 설명하는 한글 메시지를 작성하세요.
4. **자주 커밋하기**: 작업 중간에 논리적인 단위가 완료될 때마다 커밋하세요.
5. **푸시 주기**: 네트워크 상태에 따라 적절히 푸시하려면, 몇 번의 커밋마다 혹은 일정 시간마다 푸시하는 것을 고려하세요.

## Error Handling
- git 명령어가 실패하면 오류 메시지를 확인하고 적절히 조치하세요.
- 푸시 시 충돌이 발생하면 먼저 풀을 받은 후 충돌을 해결하고 다시 푸시하세요.
- 인증 오류가 발생하면 원격 저장소에 대한 접근 권한을 확인하세요.
